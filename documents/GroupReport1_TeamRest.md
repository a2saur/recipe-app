## 2\. Iteration 1 \- Sprint Retrospective

In Iteration 1, our biggest issue was getting everyone coordinated and figuring out how to stay organized with so many details involved in the project. However, through our meetings, we were able to set clearer goals and workflows and deal with questions and issues as they arose. For example, we established a github workflow where when someone works on an issue, they create a branch with the appropriate name, and when they are done, they create a pull request which someone else will review. For the next iteration, we plan to stick to this workflow and update our to-dos more frequently.
